# AI DApp + 游戏币系统平台

本仓库用于构建一个「面向普通用户」的 AI DApp 平台：前端体验接近 Web2 应用，通过游戏币（虚拟币）完成计费与激励；底层保留演进为 Web3 DApp 的扩展空间。

本项目只关注「平台与游戏币系统」本身，AI 模型推理将由**独立的 Python AI 服务项目**提供 HTTP 接口，本仓库通过调用该接口完成计费和转发。

> 说明：当前只初始化文档与设计，不包含具体代码实现。

---

## 核心目标

- 用户体验接近 Web2：不需要钱包、不懂代币也能直接使用 AI
- 支持游戏币扣费、奖励、任务等游戏化机制
- 支持从「完全链下积分」到「链下 + 链上混合」再到「完全链上托管」的平滑升级
- AI 推理能力由独立的 Python 服务（如 FastAPI）提供，本项目只做平台与计费

---

## 技术架构概要

- 前端：Web（推荐 React / Next.js + TailwindCSS）
- 平台后端：Python（推荐 FastAPI）
  - 用户系统与登录
  - 链下游戏币余额与交易流水
  - AI 调用计费与转发（调用外部 AI 服务）
  - 任务 / 奖励 / 商城逻辑（后续扩展）
- AI 服务后端：独立项目
  - 负责真正的模型推理（对话 / 生成 / 图像等）
  - 暴露统一接口供本平台调用
- 可选 Web3 层（非 MVP 必选）
  - ERC20 代币合约
  - 支付合约
  - 托管钱包 / AA 方案

架构详细说明见 `docs/ARCHITECTURE.md`。

## MVP 范围（本仓库关注部分）

MVP 阶段，本项目重点完成：

- 链下游戏币系统
  - 用户表与余额字段
  - 交易流水（充值、奖励、扣费）
  - 初始赠币与任务奖励策略
- 平台后端基础能力
  - 用户注册 / 登录 / 认证
  - 查询余额与交易记录
  - AI 调用扣费逻辑（按调用次数或 Token 数量计价）
  - 调用外部 AI 服务接口并转发响应
- 前端基础页面
  - 登录 / 注册页
  - AI 对话页
  - 游戏币钱包页（展示余额与流水）
  - 简单任务 / 奖励页

AI 模型本身的实现、训练与部署由「AI 服务项目」负责，本仓库只负责调用它。
